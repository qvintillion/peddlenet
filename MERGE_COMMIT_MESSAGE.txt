═══════════════════════════════════════════════════════════════════
🔧 SERVER-SIDE CHANGES (signaling-server.js)
═══════════════════════════════════════════════════════════════════

1. INCREASED TIMEOUT TOLERANCES
   - pingTimeout: 60s → 90s (more tolerant of mobile/Cloud Run latency)
   - pingInterval: 25s → 35s (reduces false disconnects)
   - connectTimeout: NEW 60s (allows time for cold starts)
   - upgradeTimeout: NEW 30s (polling → websocket upgrade)
   Lines: 131-134

2. AUTOMATIC MEMORY CLEANUP SYSTEM
   - Runs every hour automatically
   - Public rooms: Deletes messages >24 hours old
   - All rooms: Limits to last 100 messages
   - Inactive users: Removed after 24 hours offline
   - Activity log: Trimmed to last 1000 entries
   - P2P connections: Cleaned for disconnected sockets
   - Graceful shutdown handling (SIGTERM/SIGINT)
   Lines: 2019-2105

3. COLD START DETECTION
   - Detects first 30 seconds after server start
   - Adaptive timeouts: 45s for cold starts, 30s normal
   - Logs cold start connections with ❄️ indicator
   - SERVER_START_TIME tracking
   Lines: 27-47

4. CONNECTION HEALTH MONITORING
   - Health pings every 35 seconds
   - Tracks ping/pong statistics per connection
   - Forces disconnect after 3 missed pongs
   - Logs connection duration and stats on disconnect
   Lines: 1532-1576, 1916-1931

5. CORS CONFIGURATION FIX
   - Added Vercel preview domain support using regex patterns
   - Patterns: /^https:\/\/festival-chat-.*\.vercel\.app$/
   - Patterns: /^https:\/\/.*\.vercel\.app$/
   - Socket.IO requires regex (not wildcard strings) for dynamic subdomains
   - Commented out unused Firebase preview patterns
   Lines: 91-114

6. VERSION UPDATE
   - Updated from: 1.1.0-admin-enhanced
   - Updated to: 1.2.0-phase1-optimized
   - Locations: lines 368, 422, 673, 1517

💻 CLIENT-SIDE CHANGES
═══════════════════════════════════════════════════════════════════

1. MATCHING SERVER TIMEOUTS (src/hooks/use-websocket-chat.ts)
   - timeout: Minimum 20s (handles Cloud Run cold starts)
   - connectTimeout: 30s
   - pingTimeout: 90s (matches server)
   - pingInterval: 35s (matches server)
   Lines: 318-333

2. AUTOMATIC RECONNECTION ENABLED (src/hooks/use-websocket-chat.ts)
   - reconnection: false → true
   - reconnectionAttempts: 0 → 5 (try up to 5 times)
   - reconnectionDelay: 1000ms (start delay)
   - reconnectionDelayMax: 5000ms (exponential backoff cap)
   - randomizationFactor: 0.5 (jitter to prevent thundering herd)
   Lines: 322-327

   WHY: Handles "io server disconnect" when Cloud Run scales to zero.
   Clients now automatically reconnect when server wakes up.

3. ENVIRONMENT DETECTION FIX (src/utils/server-utils.ts)
   - Previously: All .vercel.app domains classified as "production"
   - Now: Vercel preview deployments correctly detected as "staging"
   - Preview pattern: /^festival-chat-[\w-]+\.vercel\.app$/
   - Production: Only peddlenet.app custom domain
   Lines: 216-233

   WHY: Vercel preview deployments were incorrectly showing "production"
   in logs, causing confusion during testing.
